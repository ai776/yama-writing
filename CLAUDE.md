# 書籍量産システム — tldvナレッジ × Claude Code

テーマを入力するだけで、tldvミーティング文字起こし（約2,900件）をナレッジベースとして
書籍を自動制作するClaude Code Skills パッケージ。

## クイックスタート

```
/book-factory AI活用で売上を3倍にする方法
```

これだけで以下が自動実行される:
1. tldvフォルダからテーマに関連するナレッジを自動検索
2. BookSpec（戦略＋制作仕様）を自動生成
3. 目次・見出し・章設計を自動構成
4. 章ごとに本文を執筆
5. 図表生成・エクスポート

## フォルダ構成

```
.
├── CLAUDE.md              # このファイル
├── tldv/                  # ナレッジベース（約2,900件のミーティング文字起こし）
├── output/                # 書籍出力先（テーマごとにサブフォルダ）
├── review/                # レビュー済み原稿
├── rules/                 # Cursorルール（旧、参考用）
├── .claude/skills/        # Claude Code Skills（書籍制作パイプライン）
└── ◆ AI生成文章の特徴と執筆のポイント.md  # AI表現回避ガイド
```

## tldvフォルダについて

- ミーティングの文字起こしテキスト（.txt）
- 形式: `[タイムスタンプ] 話者名\n発言内容`
- テーマに関連するファイルを自動検索し、ナレッジとして活用する
- **このフォルダのファイルは情報源であり、編集しないこと**

## 利用可能なスキル

### メインコマンド（これだけ覚えればOK）

| コマンド | 説明 |
|---------|------|
| `/book-factory テーマ名` | テーマから書籍を自動制作（推奨） |

### 個別スキル（上級者向け）

| コマンド | 説明 |
|---------|------|
| `/book-producer` | 手動オーケストレーション（ヒアリングあり） |
| `/book-strategy` | 戦略設計のみ |
| `/book-spec` | 制作仕様設計のみ |
| `/book-ingest` | 素材整形のみ |
| `/book-knowledge` | ナレッジ構造化のみ |
| `/book-structure` | 全体構成設計のみ |
| `/book-title` | タイトル生成のみ |
| `/book-draft` | 本文執筆のみ |
| `/book-review` | 原稿レビューのみ |
| `/book-rewrite` | リライトのみ |
| `/book-qa` | 品質チェックのみ |
| `/book-consistency` | 章間矛盾チェックのみ |
| `/book-export` | Docx/PDF出力のみ |

## 重要ルール

- **情報源厳守**: tldvナレッジにない事実・エピソードは絶対に使わない
- **AI表現回避**: 「◆ AI生成文章の特徴と執筆のポイント.md」のNG表現リストを厳守
- **本文に出典を入れない**: 引用元の記載は提出レポートにのみ記載
- **承認ゲート**: 4箇所でユーザー確認を行う（ナレッジ選定/BookSpec/構成/完成原稿）
