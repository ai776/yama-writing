# 見出し設計ガイドライン

## 1. 階層構造ルール

### H1（章タイトル）
- 読者の期待を設定する「約束」の役割
- 章の核心を一文で伝える
- 全章で統一されたトーン（疑問形で揃える / 名詞止めで揃える 等）
- 15〜30文字が目安

### H2（節タイトル）
- 章の論理的な区切りを示す
- 1章あたり3〜6個が適正（多すぎると散漫、少なすぎると密度が高すぎる）
- 読者が節タイトルだけ拾い読みしても章の流れがわかること
- 10〜25文字が目安

### H3（小項目）
- 必須ではない。節の中で更に分割が必要な場合のみ
- ハウツー・手順・チェックリストに適する
- 使うなら2〜4個で揃える
- 10〜20文字が目安

## 2. 論理接続パターン

見出し間の接続に以下のパターンを意識する:

| パターン | 説明 | 例 |
|---------|------|-----|
| 因果 | AだからB | 「なぜ太るのか」→「リセットの仕組み」 |
| 時系列 | 前→後 | 「食べすぎた夜」→「翌朝やること」 |
| 対比 | AとBの違い | 「やるべきこと」→「やらなくていいこと」 |
| 具体化 | 概要→詳細 | 「3つのポイント」→「ポイント①〜③」 |
| 積み上げ | 基礎→応用 | 「基本ストレッチ」→「応用メニュー」 |
| 問いと答え | 問い→回答 | 「なぜ効くの？」→「理由はこれだ」 |

## 3. 読者理解順序

1. **知りたいこと** — まず読者の疑問・悩みに触れる
2. **理由・背景** — なぜそうなのかを説明する
3. **方法・手順** — 具体的にどうするかを示す
4. **事例・証拠** — うまくいった例を見せる
5. **まとめ・行動** — 次のアクションを促す

この順序を各章内の見出し設計に反映させる。

## 4. 命名規則

### 良い見出しの特徴
- 具体的（抽象語を避ける）
- 読者が「読みたい」と思える
- 情報量がある（「ポイント」だけでなく「寝る前5分のポイント」）
- 動詞 or 数字を含む

### 避けるべき見出し
- 「はじめに」「おわりに」（章内の節タイトルとしては不可）
- 「まとめ」の連発（章末に毎回「まとめ」は単調）
- 「〜について」（情報量ゼロ）
- 「〜の重要性」（AI臭い）

## 5. 章間整合チェックリスト

- [ ] 全章のH1が同じトーンで統一されているか
- [ ] H2の粒度が章ごとに大きく異ならないか
- [ ] 同じ表現・パターンが連続していないか
- [ ] 核心トピック（`*`）が各章に少なくとも1つあるか
- [ ] 見出しだけ読んで書籍全体のストーリーが理解できるか
