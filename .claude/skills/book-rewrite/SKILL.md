# book-rewrite — ⑪ リライト（修正指示反映）

修正指示書に基づき、既存原稿を章ごとにリライトする。
book-draft（⑨）の後工程として位置付けられ、編集者・校閲者・AIレビューからの
フィードバックを原稿に反映する専門スキル。

## 役割

- 修正指示書（レビューシート）の解析・分類
- 指示に基づく原稿の章ごとリライト
- リライト時の AI 表現回避（book-qa の基準を遵守）
- 修正の根拠・変更箇所の透明なレポート生成
- 原稿の「魂」（著者のトーン・主張の核心）を維持したまま改善

## Inputs / Outputs

- **In:**
  - 既存原稿（`{書籍タイトル}.md`）
  - 修正指示書（`{書籍タイトル}_Review{NN}.md` 等）
  - `BookSpec.json`（存在する場合）
- **Out:**
  - リライト済み原稿（`{書籍タイトル}_v{N}.md`）
  - リライトレポート（`rewrite_report.md`）

## ワークフロー

1. **修正指示書の読み込みと分類**
   - 修正指示書を全文読み込む
   - 各指示を以下の種別に分類:

   | 種別 | 説明 | 例 |
   |------|------|----|
   | 論理修正 | 論理の飛躍・矛盾の解消 | 「楽観→戦略に昇華」 |
   | データ修正 | 数値・事実の正確性向上 | 「経費率15%→20〜25%」 |
   | コンプラ修正 | 法的・倫理的リスク回避 | 「フルローン表現の撤廃」 |
   | 構成修正 | 重複削除・章間役割の明確化 | 「第2章はソフト面、第3章はハード面」 |
   | 表現修正 | 文体・トーンの調整 | 「断定→マイルド表現」 |
   | 新規追加 | コラム・図表・注釈の追加 | 「デッドクロスのコラム追加」 |

2. **リライト方針の確認（G11-H: 人間承認）**
   - 分類結果と適用方針を一覧表で提示
   - 「そのまま適用」「調整して適用」「不採用」を指示ごとに人間が判断
   - 承認されるまでリライトに着手しない

3. **章ごとリライト実行**
   - 承認された方針に基づき、章単位でリライト
   - 各修正箇所に対して:
     - 原文を引用
     - 修正後の文章を記載
     - 修正理由を明記
   - リライト時のルール:
     - **修正指示の「修正原稿案」がある場合**: 原稿案をベースにしつつ、前後文脈に合わせて調整
     - **修正指示が方針のみの場合**: 著者のトーンを維持して新規執筆
     - **情報源にない事実の捏造は絶対禁止**
     - **AI 表現を使わない**（book-qa の banned_phrases.md / style_manual.md を遵守）

4. **リライトレポート生成**
   - 全修正箇所の一覧（修正前→修正後）
   - 適用した指示と不採用にした指示
   - 章ごとの文字数変動
   - AI 表現セルフチェック結果

5. **3段階レビュー**
   - book-qa に提出（G11-Q: リライト品質の校閲）
   - book-producer に提出（G11-P: 全体整合性の確認）
   - 人間に提出（G11-H: 最終承認）

## book-qa との連携

book-rewrite の出力は book-qa（品質保証層）のフルチェック対象となる。
book-qa は以下を重点的にチェック:

- 修正指示が正しく反映されているか
- リライトで新たな AI 表現が混入していないか
- 修正によって新たな論理矛盾が生じていないか
- 著者のトーン・パーソナリティが維持されているか
