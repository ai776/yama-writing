# book-factory — テーマ入力→書籍自動制作

tldvフォルダのミーティング文字起こしをナレッジベースとして、
テーマを指定するだけで書籍を自動制作するワンストップスキル。

## 使い方

```
/book-factory AI活用で売上を3倍にする方法
```

または

```
/book-factory
テーマ: 不用品回収ビジネスの始め方
ペルソナ: 副業を探している30代サラリーマン
```

## Inputs / Outputs

- **In:** テーマ（必須）、ペルソナ（任意）、その他オプション
- **Out:** `output/{テーマスラッグ}/` に書籍一式を出力

## 出力フォルダ構成

```
output/{テーマスラッグ}/
├── BookSpec.json          # 戦略＋制作仕様（統合版）
├── StrategySpec.md        # 戦略設計
├── ProductionSpec.md      # 制作条件
├── cleaned_transcript.md  # tldvから抽出・整形したナレッジ
├── knowledge_base.md      # 構造化ナレッジ
├── toc.md                 # 目次構成案
├── chapter_budget.md      # 文字数予算
├── chapter_blueprints.md  # 章設計図
├── heading_spec.md        # 見出し体系
├── {書籍タイトル}.md       # 本文（完成原稿）
└── figures/               # 図表
```

## 自動制作フロー

### Phase 0: tldvナレッジ検索（自動）

1. テーマに関連するキーワードを生成（5〜10個）
2. `tldv/` フォルダ内の全テキストファイルをキーワード検索
3. 関連度スコアで上位ファイルを選定（最大20ファイル）
4. 選定ファイルの関連セグメントを抽出
5. **→ ユーザー確認:** 「これらのナレッジを使用します。OKですか？」

### Phase 0.5: 情報充足チェック＆補足ヒアリング（自動）

Phase 0で選定したナレッジを分析し、テーマに必要な具体情報が不足していないか確認する:

1. テーマのタイトルが約束する内容（例:「AI×外注で売上の規模をつくる」）に対して、tldvナレッジが十分な具体情報を持っているか判定
2. 不足がある場合（例: 具体的なツール名、プラットフォーム名、報酬体系、契約形態など）、**AIが勝手に補完せず、ユーザーに質問する**
3. 質問例:
   - 「タイトルが『AI×外注で～』ですが、具体的なAIツール名が会議録にありません。補足いただけますか？それとも抽象的に書きますか？」
   - 「外注の報酬体系・契約形態の情報がありません。この部分はカットしますか？」
4. ユーザーの回答を情報源として追加し、Phase 1へ進む
5. ユーザーが「なしで進めて」と言った場合は、該当の具体情報に触れない構成に調整する

### Phase 1: BookSpec自動生成（ヒアリング省略）

以下をテーマから自動推論し、BookSpecを一括生成:

| 項目 | 自動推論ルール |
|------|---------------|
| 事業ゴール | テーマから推測（コンサル誘導/セミナー集客/権威性構築） |
| ペルソナ | テーマのターゲット層を推測（ユーザー指定があればそれを優先） |
| 核心メッセージ | tldvナレッジ内の最頻出主張から抽出 |
| トーン | 山本さんの口調をtldvから分析して設定 |
| 分量 | デフォルト: 5万字 / 7章構成 |
| 情報源タイプ | `tldv`（固定） |

**→ ユーザー確認:** 「BookSpecを生成しました。修正点はありますか？」

### Phase 2: ナレッジ構造化（自動）

1. Phase 0で選定したtldvファイルを `cleaned_transcript.md` に結合
2. **book-knowledge** でタグ付き構造化ナレッジを生成
3. テーマに無関係なセグメントを除外

### Phase 3: 構成設計（自動）

1. **book-structure** で目次＋文字数予算を生成
2. **book-title** でタイトル候補を3つ生成
3. **book-chapter-blueprint** で各章の設計図を作成
4. **book-heading** で見出し体系を設計
5. **→ ユーザー確認:** 「目次とタイトルを確認してください」

### Phase 4: 本文執筆（自動）

1. **book-draft** で章ごとに執筆
2. 各章は book-qa でセルフチェック
3. 全章完成後、**book-consistency** で矛盾チェック
4. **→ ユーザー確認:** 「原稿が完成しました。レビューしてください」

### Phase 5: 仕上げ（自動）

1. **book-figdesign** + **book-figgen** で図表生成
2. **book-export** でDocx/PDF出力（pandoc利用可能な場合）

## tldvナレッジ検索ルール

### キーワード生成

テーマから以下の3種類のキーワードを自動生成:

1. **直接キーワード** — テーマの中核語（例: 「AI活用」→「AI」「活用」「自動化」）
2. **関連キーワード** — テーマの周辺語（例: 「AI活用」→「ChatGPT」「効率化」「DX」）
3. **人物キーワード** — テーマに詳しそうな人物名（tldvファイル名から推測）

### 関連度スコアリング

各tldvファイルに対して:
- キーワード出現回数 × 重み（直接=3, 関連=1, 人物=2）
- テーマとの意味的関連度（ファイル冒頭500文字で判定）
- スコア上位20ファイルを選定

### セグメント抽出

選定ファイルから:
- テーマに関連する発言ブロックのみ抽出
- 前後の文脈（2発言分）も含める
- タイムスタンプを保持

## ユーザー確認ポイント（承認ゲート）

量産モードでも品質担保のため、以下の4箇所でユーザー確認を行う:

| ゲート | タイミング | 確認内容 |
|--------|-----------|---------|
| G-0 | tldv検索後 | 使用するナレッジファイル一覧 |
| G-0.5 | 情報充足チェック後 | テーマに必要な具体情報の不足箇所＋補足質問 |
| G-1 | BookSpec生成後 | 戦略＋制作仕様＋ターゲット読者の確認 |
| G-3 | 構成設計後 | 目次＋タイトル＋見出し＋タイトルと内容の温度差チェック |
| G-4 | 原稿完成後 | 完成原稿のレビュー |

ユーザーが「全部自動で」と指示した場合は、G-0〜G-3を省略してG-4のみ実施。

## デフォルト設定（カスタマイズ可能）

```json
{
  "defaults": {
    "total_chars": 50000,
    "chapters": 7,
    "tone": "プロフェッショナルかつ親しみやすい（山本さんの口調）",
    "forbidden_expressions": "AI表現回避リスト準拠",
    "figure_policy": "章ごとに1〜2点",
    "cta_type": "コンサル/セミナー誘導",
    "output_dir": "output/"
  }
}
```
