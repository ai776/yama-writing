# フィードバック分類・優先度判定ルール

## 1. フィードバック形式の対応

| 形式 | 処理方法 |
|------|---------|
| 修正ポイント原稿（マークアップ付き） | マークアップを修正指示に変換 |
| レビューコメント（箇条書き） | 章・箇所への紐付けを追加 |
| メール・チャットのテキスト | 修正指示を抽出・構造化 |
| 音声メモ文字起こし | book-ingest のルールに準じてクリーニング後、修正指示を抽出 |
| 校正記号入り原稿 | 校正記号を修正指示に変換 |

## 2. 分類カテゴリ

### 全体方針（GLOBAL）
- トーン・文体の変更指示
- 読者層の見直し
- 核心メッセージの修正
- 章構成の変更

### 章単位（CHAPTER）
- 章の論理展開の修正
- 内容の追加・削除
- 章内の構成変更
- 情報の補足要求

### 箇所単位（POINT）
- 特定の文・段落の修正
- 表現の変更
- 数値の修正
- 事実関係の修正

### 形式（FORMAT）
- 誤字脱字
- 表記統一
- フォーマット修正

## 3. 優先度判定基準

### MUST（必ず修正）
- 事実誤認（数値の間違い、法律の誤り等）
- 論理破綻（前後で矛盾する主張）
- 著者の明確な修正指示（「ここは変えてください」）
- コンプライアンス問題（差別表現、名誉毀損リスク等）

### SHOULD（強く推奨）
- 読みやすさの改善（冗長な文、分かりにくい説明）
- 論調の調整（批判が強すぎる、煽りすぎる）
- 情報の追加要求（「ここに具体例が欲しい」）
- Blueprint の禁止事項に抵触する箇所

### COULD（あれば良い）
- 好みの範囲の表現変更
- 軽微な言い回し調整
- 参考情報の追加提案

## 4. 修正意図の三点セット

各修正指示は以下の三点セットで記述する:

```
【何を】第3章 第2節 L.145「利回り8%以上の物件を探しましょう」
【なぜ】数値が第6章の記述（6%台でもOK）と矛盾するため
【どう直すか】「基本的には利回り8%以上を目安にしますが…」に修正
```

## 5. 重複・矛盾の処理

### 同一箇所への複数指摘
- 修正方向が同じ → 統合（最も具体的な指摘を採用）
- 修正方向が異なる → `<!-- CONFLICT -->` タグ付きで両方記録、著者判断を仰ぐ

### フィードバック内の矛盾
- 「もっと具体的に」と「冗長」が同じ箇所にある場合等
- 矛盾を明示し、著者に方向性を確認する
