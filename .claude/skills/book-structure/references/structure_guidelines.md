# 全体設計ガイドライン

## 1. 論理構造の設計

### 章配置の論理パターン（基本5型）

| パターン | 説明 | 適する書籍 |
|---------|------|-----------|
| 問題→原因→解決 | 悩み提示 → なぜ → どうする | ハウツー系 |
| 基礎→応用→実践 | 知識 → 活用法 → やってみよう | 教育系 |
| ストーリー型 | 挫折 → 発見 → 変化 | 体験記系 |
| ステップ型 | Step1 → Step2 → Step3 | 手順系 |
| 対比型 | Before → After / 従来 → 新しい | 提案系 |

### 拡張構成パターン（実務で高頻度）

#### 権威性証明型（専門家ブランディング向け）

```
第1章: 業界の常識を覆す主張（衝撃の導入）
第2章: なぜ従来のやり方では失敗するのか（問題の深掘り）
第3章: 著者独自のメソッド/理論の全体像
第4〜6章: メソッドの各要素を具体事例で解説
第7章: 実践者の成果・変化（社会的証明）
第8章: 読者が今日から始められること + CTA
```

**ポイント:** 第1章で「この人の話は聞く価値がある」と思わせる。著者の独自性を軸に構成。
**適する役割:** 権威性証明、コンサル販売、セミナー集客

#### 恐怖→安心型（リスク回避訴求向け）

```
第1章: 読者が気づいていない危険の提示（恐怖喚起）
第2章: 実際に失敗した人の事例（他人事ではない感）
第3章: なぜ多くの人が同じ失敗をするのか（原因分析）
第4章: 正しい判断基準の提示（著者の専門性）
第5〜7章: 具体的な防衛策・実践法
第8章: 安心して行動するためのロードマップ + CTA
```

**ポイント:** 第1〜2章で「このままではまずい」と感じさせ、第3章以降で「この人に頼れば大丈夫」に転換。
**適する役割:** リード獲得、個別相談誘導、診断サービス販売

#### 変革ジャーニー型（自己啓発・ライフスタイル変革向け）

```
第1章: 著者の過去の苦しみ（読者との共感接点）
第2章: 転機となった出来事（ターニングポイント）
第3章: 変化のきっかけとなった考え方の転換
第4〜6章: 具体的な実践と試行錯誤（リアルなプロセス）
第7章: 変化後の世界（ビジョンの提示）
第8章: 読者自身の変革への招待 + CTA
```

**ポイント:** 著者の体験に読者が自己投影し、「自分もできるかも」と思わせる。
**適する役割:** コミュニティ誘導、講座販売、ブランド構築

#### Q&A/お悩み解決型（実用書向け）

```
第1章: 「こんな悩み、ありませんか？」（問題の網羅）
第2〜7章: 各悩みに対する回答（1章1テーマ、独立読み可能）
第8章: まとめ + よくある質問 + CTA
```

**ポイント:** 各章が独立して読めるため、目次を見て「自分に関係ある章」だけ読む読者にも対応。
**適する役割:** 教育コンテンツ、営業資料代替

#### 二項対立型（常識破壊向け）

```
第1章: 「○○は間違いだった」（常識への挑戦）
第2章: 従来の常識が生まれた背景
第3章: 新しい視点の提示（著者の主張）
第4〜6章: 新旧の比較を具体事例で展開
第7章: 新しいアプローチで成功した事例
第8章: パラダイムシフトへの招待 + CTA
```

**ポイント:** 第1章で「え、そうなの？」という驚きを作り、最後まで読ませる牽引力を生む。
**適する役割:** 権威性証明、セミナー集客、ブランド構築

### パターン選定の判断基準

| 判断軸 | 選ぶべきパターン |
|--------|----------------|
| 書籍の役割が「リード獲得」 | 恐怖→安心型 or 権威性証明型 |
| 書籍の役割が「権威性証明」 | 権威性証明型 or 二項対立型 |
| 書籍の役割が「ブランド構築」 | 変革ジャーニー型 or 二項対立型 |
| 書籍の役割が「教育コンテンツ」 | Q&A型 or 基礎→応用→実践 |
| 書籍の役割が「営業資料代替」 | Q&A型 or 問題→原因→解決 |
| 情報源がセミナー文字起こし | ストーリー型 or 変革ジャーニー型（語り口を活かせる） |
| 情報源がYouTube動画 | Q&A型 or ステップ型（動画の独立性を活かせる） |
| 情報源が既存原稿リボーン | 元の構成を活かしつつ、役割に合ったパターンに再構成 |

### 構成パターンの組み合わせ

単一パターンに固執する必要はない。たとえば:
- **前半: 恐怖→安心型 + 後半: ステップ型** = 問題認知させてから具体的手順を示す
- **全体: 権威性証明型 + 各章内: 対比型** = メソッド紹介を新旧比較で展開

組み合わせる場合は、**読者の感情の流れが途切れないこと** を最優先に判断する。

### 論理構造チェック

- [ ] 第1章で読者の問題意識を十分に喚起しているか
- [ ] 各章は前章の結論を前提としているか（積み上がっているか）
- [ ] 「読者が知りたい順」になっているか（著者が書きたい順になっていないか）
- [ ] 結論・行動喚起が最終章に配置されているか
- [ ] 選択した構成パターンがBookSpecの「書籍の役割」と整合しているか
- [ ] 構成パターンの選定理由を明示できるか

## 2. ストーリーフロー設計

読者の感情・理解の流れを意識した章配置:

```
共感（あなたの悩みわかります）
  ↓
発見（実はこういう仕組みです）
  ↓
納得（だからこうすればいいんです）
  ↓
実践（具体的にはこうします）
  ↓
確信（やってみたら変わりました）
  ↓
行動（さあ、始めましょう）
```

各章がこのフローのどこに位置するかを明示する。

## 3. 文字数配分ルール

1. 核心トピック（`*` マーキング）を含む章には 10〜20% 多めに配分
2. 導入章（第1章）は読者の興味を引くため、やや多めに設定可
3. 最終章は行動喚起を含むため、あとがきとのバランスを考慮
4. 各章の文字数は ±500字 の範囲で調整可能
5. まえがき・あとがきは全体の5〜10%ずつ

## 4. 情報源のカバレッジ

- cleaned_transcript.md の全トピックが目次のいずれかの章に割り当てられていること
- 割り当てられないトピックは「対象外」として明示（理由付き）
- 重複テーマ（`<!-- OVERLAP -->`）は最も適切な1箇所に配置
- 矛盾箇所（`<!-- CONFLICT -->`）は本文内で両論併記 or 著者に判断を仰ぐ
