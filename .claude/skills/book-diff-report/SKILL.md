# book-diff-report — 修正箇所一覧（Diff Report）生成

初稿と修正稿を比較し、ライター・編集者に共有するための修正箇所一覧を生成する。
book-rewrite（⑪）の後工程として、修正内容の可視化・共有を担当する。

## 役割

- 初稿と修正稿の差分検出（段落単位）
- 修正箇所の章ごとグルーピング
- 各修正箇所への修正意図の付与（修正指示書があれば参照）
- ライター・編集者向けの共有可能な一覧生成

## Inputs / Outputs

- **In:**
  - 初稿（`{書籍タイトル}.md`）
  - 修正稿（`{書籍タイトル}_v{N}.md` または Second-draft の原稿）
  - 修正指示書（`{書籍タイトル}_Review{NN}.md`）（任意・修正意図の補強に使用）
- **Out:**
  - 修正箇所一覧（`修正箇所一覧_{稿名}.md`）

## ワークフロー

1. **原稿の読み込みと構造解析**
   - 初稿・修正稿を全文読み込む
   - 見出し（`#`, `##`, `###`）で章・節に分割
   - 段落単位で比較可能なブロックに分解

2. **差分検出**
   - 章ごとに段落を比較
   - 変更種別を判定:

   | 種別 | 説明 |
   |------|------|
   | 修正 | 既存テキストの変更 |
   | 追加 | 新規テキスト・節の追加 |
   | 削除 | テキスト・節の削除 |
   | 移動 | 別の位置への移動 |

3. **修正意図の付与**
   - 修正指示書（Review）が存在する場合:
     - 各差分を修正指示と照合
     - 対応する指示の「編集者コメント」を修正意図として引用
   - 修正指示書がない場合:
     - 変更内容から修正意図を推定して記載

4. **一覧生成**
   - [templates/diff_report.md](templates/diff_report.md) のフォーマットに従って出力
   - 章ごとにグルーピング
   - 通し番号を付与

5. **人間承認**
   - 生成した修正箇所一覧を人間に提示
   - 承認後、ライター・編集者に共有

## 出力フォーマット例

各修正箇所は以下の形式:

```markdown
### 修正{N}: {章名} — {修正箇所の概要}

**修正指示:** {対応する修正指示番号（あれば）}

**旧（初稿）:**
> {修正前のテキスト}

**新（第二稿）:**
> {修正後のテキスト}

**修正意図:**
{なぜこの修正を行ったかの説明}
```
