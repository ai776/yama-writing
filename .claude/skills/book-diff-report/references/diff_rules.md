# 差分検出・分類ルール

book-diff-report が差分検出時に従うルール。

---

## 1. 比較単位

- **段落単位**で比較する（空行区切り）
- 見出し行（`#`で始まる行）は構造マーカーとして扱い、章・節の境界を判定する
- 箇条書き・表は1ブロックとして扱う

## 2. 差分の分類基準

| 分類 | 判定条件 |
|------|---------|
| **軽微な修正** | 語尾の変更、助詞の修正、句読点の追加・削除 |
| **表現修正** | 文意を変えずに表現を変更（断定→マイルド化など） |
| **内容修正** | 数値・事実・データの変更、情報の追加・削除 |
| **構成修正** | 段落の並び替え、節の追加・削除・統合 |
| **コンプラ修正** | 法的リスク・クレームリスクのある表現の修正 |

## 3. 一覧に含めるもの / 含めないもの

### 含める
- 文意に影響する修正（表現修正以上）
- 数値・データの変更
- 新規追加された段落・節
- 削除された段落・節
- コンプライアンス関連の修正

### 含めない（原則）
- 誤字脱字の修正（ただし固有名詞の修正は含める）
- 句読点のみの変更
- 改行位置の変更
- Markdownフォーマットのみの変更（`**太字**` 追加など）

※ ただし、ライターへの共有目的で「すべての変更を見せたい」場合は、
  軽微な修正もまとめて末尾に「その他の軽微な修正」として記載可。

## 4. 修正意図の記述ルール

- 修正指示書（Review）の指示番号と紐づける（例：「修正指示⑤に対応」）
- 修正指示にない独自の改善は「編集判断による修正」と明記
- 修正意図は1〜2文で簡潔に記述する
- 主語は「編集者」または「校閲者」（著者に向けた文体）

## 5. 出力時の注意

- 章の順番は原稿の章順に従う
- 同一章内では修正箇所の出現順に並べる
- 通し番号は全体で一意（章ごとにリセットしない）
- 長い引用は先頭50文字程度 + 「...」で省略可
