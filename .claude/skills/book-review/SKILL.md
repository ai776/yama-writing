# book-review — 原稿レビュー・修正指示書生成

完成原稿を編集者視点でレビューし、修正指示書を生成する。
book-draft（⑨）の後工程、book-rewrite（⑪）の前工程。

## 役割

- 完成原稿の多角的レビュー（コンプラ・データ・論理・構成・表現）
- 修正指示書の生成（修正原稿案付き）
- 出版時期・市況に応じたリスク判断
- 章間整合性の確認
- 読者ペルソナの視点からの実用性評価

## Inputs / Outputs

- **In:**
  - 完成原稿（`{書籍タイトル}.md`）
  - `BookSpec.json`（存在する場合）
  - 出版時期・市況情報（任意）
- **Out:**
  - 修正指示書（`{書籍タイトル}_Review{NN}.md`）

## ワークフロー

1. **原稿の全文読み込みと構造把握**
   - 全章の見出し・文字数・論点を一覧化
   - BookSpec があれば戦略・ペルソナとの整合を確認

2. **5軸レビュー実施**

   | 軸 | チェック内容 | 重大度 |
   |----|------------|--------|
   | コンプライアンス | 法的リスク・クレームリスク・断定表現 | 最高 |
   | データ正確性 | 数値の妥当性・シミュレーションの前提・経費率 | 高 |
   | 論理整合性 | 論理の飛躍・矛盾・章間の不整合 | 高 |
   | 構成 | 重複・役割分担・章間のブリッジ | 中 |
   | 表現 | トーン・断定度合い・地域配慮・AI臭さ | 中 |

3. **修正指示の起案**
   - 各指示に通し番号（①②③...）を付与
   - 章ごとにグルーピング
   - 各指示に以下を含める：
     - **種別**（コンプラ修正/データ修正/論理修正/構成修正/表現修正/新規追加）
     - **対象箇所**（章・ページ・見出し）
     - **原文引用**
     - **修正原稿案**（具体的な修正後文章）または**修正方針**
     - **編集者コメント**（なぜこの修正が必要かの根拠）

4. **全体戦略と編集方針の策定**
   - 原稿の最大の強み・弱点を特定
   - 編集方針を1文で要約（例：「楽観を戦略に昇華させる」）

5. **修正指示書のフォーマット出力**
   - [templates/review_sheet.md](templates/review_sheet.md) に従って出力

6. **人間承認**
   - 修正指示書を人間に提示
   - 承認後、book-rewrite に引き渡し

## book-rewrite との連携

book-review の出力（修正指示書）は book-rewrite の入力となる。
book-rewrite は修正指示書に基づいて原稿をリライトし、
リライトレポートを生成する。
