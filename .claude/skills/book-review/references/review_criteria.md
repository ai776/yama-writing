# レビュー基準 — 5軸の詳細判断基準

book-review Skill がレビュー時に従う判断基準。

---

## 1. コンプライアンス（重大度: 最高）

| チェック項目 | NG例 | 修正方向 |
|------------|------|---------|
| 金融商品の断定的推奨 | 「NISAはギャンブル」 | 比較・対比に留める |
| フルローンの安易な保証 | 「誰でもフルローンが出る」 | 「条件が整えば可能」に |
| 利回り・収益の確約表現 | 「必ず儲かる」 | 「シミュレーション上は」に |
| 地域・属性への差別的表現 | 「横浜は山と坂道」 | 事実を維持しマイルドに |
| 他商品の不当な貶め | 「株はダメ」 | 比較優位として論じる |

**原則：** 読者が書籍の通りに行動して失敗した場合、「本に嘘が書いてあった」とクレームになりうる表現はすべて修正対象。

## 2. データ正確性（重大度: 高）

| チェック項目 | 確認方法 |
|------------|---------|
| 経費率の妥当性 | 業界標準（20〜25%）との乖離を確認 |
| シミュレーションの前提 | 金利・空室率・修繕費が現実的か |
| 数値の章間整合 | 同じ数値が全章で統一されているか |
| 時代性 | 「現在」の融資環境・建築コストと一致するか |
| 出典の有無 | 公的データには出典を求める |

**原則：** 楽観的すぎる数値は読者の破綻につながる。厳しめの数字を見せることが優しさ。

## 3. 論理整合性（重大度: 高）

| チェック項目 | NG例 |
|------------|------|
| 論理の飛躍 | 前提なく結論に到達 |
| 自己矛盾 | 「初心者は買うな」→「自分は買った」 |
| 章間の矛盾 | 第4章のフローチャートが第8章を否定 |
| 因果関係の不備 | 原因と結果の論拠が弱い |

**原則：** 矛盾があれば、注釈・例外の明示・構成変更のいずれかで解消する。

## 4. 構成（重大度: 中）

| チェック項目 | 対応 |
|------------|------|
| 章間の重複 | 役割分担を明確にする（例：ソフト面/ハード面） |
| ブリッジの不足 | 章末に次章への接続文を追加 |
| 新規追加の必要性 | 読者が躓くポイントにコラム・図表を提案 |
| 章の順番 | 読者の理解順序に沿っているか |

**原則：** 読者が「なぜここでこの話？」と思わない構成を目指す。

## 5. 表現（重大度: 中）

| チェック項目 | 対応 |
|------------|------|
| 著者トーンの一貫性 | 一人称・語尾・口語レベルの統一 |
| 断定度合い | 「絶対」「必ず」を避ける |
| 地域・人への配慮 | 特定地域の貶めがないか |
| AI臭さ | book-qa の banned_phrases.md を参照 |

**原則：** 著者のトーンを維持しつつ、クレームリスクのある表現のみ修正。

## 修正指示の優先度

| 優先度 | 種別 | 対応速度 |
|--------|------|---------|
| 最高 | コンプラ修正 | 即座に修正 |
| 高 | データ修正 | 検証の上で修正 |
| 高 | 論理修正 | 前後章を確認して修正 |
| 中 | 構成修正 | 影響範囲を確認して修正 |
| 中 | 表現修正 | 著者トーンを崩さない範囲で修正 |
| 低 | 新規追加 | 文字数予算と相談して追加 |
