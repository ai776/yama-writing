# 運用ルール

書籍制作の全Skillに適用される共通ルール。BookSpec と合わせて「制作の憲法」となる。

## 1. 単一ファイル運用

- 原稿は **1つのMarkdownファイル** で管理する
- 目次作成後、同じファイルの下部に第1章、第2章と **追記** していく
- 別々のファイルに分割しない
- ファイル名は `{書籍タイトル}.md` とする

## 2. 情報源の厳守

- 執筆・校正するすべての文章は、著者が提供した情報源ファイルの内容 **のみ** に基づく
- ファイル内に存在しない情報、外部の知識、AIの推測や創作を付け加えることは **絶対に禁止**
- source_type ごとの制約:
  - `seminar`: 文字起こしテキストの内容のみ
  - `youtube`: 指定動画の内容のみ。動画外の知識・推測の追加は禁止
  - `reborn`: 原稿の核心（魂）は維持。新規エピソードの創作は禁止

## 3. 引用の厳格運用

- 直接引用は「」で括る
- source_type ごとの引用形式:
  - `seminar`: 発言者名、開始〜終了の行番号 or 章節名
  - `youtube`: 発言者名、動画タイトル、タイムスタンプ `[mm:ss]`
  - `reborn`: 原稿の章/節番号
- 逐語ではない要約は「要約」である旨を明記
- 複数情報源間の矛盾は、差異をそのまま記載し、どの情報源の見解かを明示

## 4. ファイル出力形式

- すべての出力は **Markdown形式** を使用
- 章タイトル: `# 第〇章 [章タイトル]`
- 小見出し: `## [小見出しタイトル]`
- 重要トピック: タイトル末尾に `*`

## 5. 役割の切り替え

- 執筆中: ブックライター（著者の代筆者）
- 最終チェック: 批判的な視点を持つ編集者
- レビュー: プロの校正者

## 6. 作業開始条件

- 著者が「実行してください」と明示するまで作業を開始しない
- BookSpec が確定していない状態で執筆に入らない
