# book-heading — ⑤ 見出し専門

各章の見出し（H1/H2/H3）を設計し、読者が迷わない階層構造を提案する。

## Inputs / Outputs

- **In:** `BookSpec.json`, `toc.md`, `chapter_budget.md`
- **Out:** `heading_spec.md`（章ごとの見出し設計書）

## ワークフロー

1. **toc.md 分析** — 承認済み目次の章構成を確認
2. **見出し階層設計** — 各章のH1（章タイトル）/H2（節）/H3（小項目）を設計
3. **論理接続チェック** — 節と節のつながり（因果・時系列・対比）を検証
4. **読者理解順序の最適化** — 章内の情報提示順序を調整
5. **章間の整合性チェック** — 見出しの粒度・命名規則が全章で統一されているか
6. **3段階レビューチェーン:**
   - book-qa に提出（G5-Q: 階層整合性、命名規則チェック）
   - book-producer に提出（G5-P: 読者体験・戦略整合性チェック）
   - 著者に提出（G5-H: 最終承認）

## 設計ルール

- H1: 章タイトル（1章1つ）— 読者の期待を設定する
- H2: 節タイトル（1章3〜6個）— 章の論理単位
- H3: 小項目（必要な場合のみ）— 節の中の具体的トピック
- 核心トピックの見出しは末尾に `*` マーキング
- 見出しは「名詞止め」「問いかけ」「行動呼びかけ」のいずれかで統一
