# 図表の種類・目的カテゴリ・選定基準

book-figdesign Skill が図表を設計する際の判断基準。

---

## 図の目的カテゴリ

各図には **必ず** 以下から主目的（＋任意で副目的）を1つ選ぶ:

| 目的 | 説明 | 適する場面 |
|------|------|-----------|
| **理解促進** | 複雑な概念・プロセスを視覚化して理解しやすくする | 専門用語の説明、手順の図解 |
| **説得** | データや比較で読者を納得させる | 効果の実証、選択肢の比較 |
| **記憶** | 要点を図で記憶に定着させる | 章のまとめ、チェックリスト |

## 図の種類一覧

| 種類 | 適する場面 | 生成方法 |
|------|-----------|---------|
| **概念図** | 抽象的な概念の構造・関係 | Nano Banana / Mermaid `mindmap` |
| **フロー** | プロセス・手順（3ステップ以上） | Mermaid `flowchart` / Nano Banana |
| **比較表** | 2つ以上の選択肢の対比 | Markdown表 / Nano Banana |
| **チェックリスト** | 行動の一覧・確認事項 | Markdown表 / Nano Banana |
| **ロードマップ** | 段階的な進行・タイムライン | Mermaid `gantt` / Nano Banana |
| **階層図** | 組織・カテゴリの包含関係 | Mermaid `mindmap` / Nano Banana |
| **シーケンス図** | やりとり・対話の流れ | Mermaid `sequenceDiagram` |
| **円グラフ** | 割合・構成比 | Mermaid `pie` |
| **イラスト指示** | 概念のイメージ化 | Nano Banana（テキスト指示） |

## 選定基準

1. **プロセス・手順** → フロー
2. **比較・対比** → 比較表
3. **構造・関係** → 概念図 / 階層図
4. **確認事項** → チェックリスト
5. **段階・成長** → ロードマップ
6. **数値データ** → 円グラフ / 比較表
7. **抽象概念** → 概念図 / イラスト指示

## 生成方法の優先順位

各図について、以下の順で生成を試みる:

1. **Nano Banana Pro プロンプト** — **全図で必須**
2. **Mermaid** — 構造化データで表現可能なら併記
3. **Markdown Table** — 表形式で表現可能なら併記

## 挿入位置ルール

- BookSpec の `visuals.placement_rule` に従う
- デフォルト: 説明の直後に配置
- 挿入位置は「`## 小見出し名` の直後」のように具体的に指定

## キャプション形式

- 20〜60文字程度の日本語
- BookSpec の `visuals.caption_style` に従う
- デフォルト: `図{章番号}-{連番}: {タイトル}`

## 図表の数の目安

- BookSpec の `visuals.frequency` に従う
- 指定がない場合: 各章に1〜2点
