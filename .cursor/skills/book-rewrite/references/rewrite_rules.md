# リライトルール — 修正適用の判断基準

book-rewrite Skill がリライト時に従うルール。

---

## 1. 修正指示の適用優先度

| 優先度 | 種別 | 対応 |
|--------|------|------|
| **最高** | コンプラ修正 | 法的リスク・クレームリスクがあるため即座に適用 |
| **高** | データ修正 | 数値の正確性は信頼性に直結。検証の上で適用 |
| **高** | 論理修正 | 論理の飛躍・矛盾は読者の離脱を招く |
| **中** | 構成修正 | 重複・役割分担の見直し。前後章への影響を確認してから適用 |
| **中** | 表現修正 | 著者トーンを崩さない範囲で適用 |
| **低** | 新規追加 | 文字数予算との兼ね合いを確認してから適用 |

## 2. リライトの絶対ルール

### やること
- 修正指示書の「修正原稿案」がある場合は、それをベースに前後文脈に合わせて調整する
- 著者のトーン・語り口・口癖を維持する（BookSpec の tone 設定を参照）
- 修正箇所の前後 500 字を必ず読み、文脈の断絶がないか確認する
- 新規追加（コラム等）は、既存の章構成の流れに自然に挿入する
- 数値の修正は、同じ数値が他章で参照されていないか全文検索して整合を取る

### やってはいけないこと
- **情報源にない事実・エピソード・数値を捏造する**
- **著者の主張の核心を変える**（表現は変えてよいが、言いたいことは変えない）
- **修正指示にない箇所を勝手にリライトする**
- **AI 表現を使う**（banned_phrases.md / style_manual.md を遵守）
- **修正原稿案を無加工でコピペする**（前後文脈に合わせた調整が必須）

## 3. 修正指示の解釈ガイドライン

### 「修正原稿案」がある場合
1. 原稿案の意図を理解する
2. 前後の文脈を確認する
3. 著者のトーンに合わせて原稿案を調整する
4. 調整理由をレポートに記載する

### 「方針のみ」の場合（具体的な文章がない）
1. 方針の意図を正確に理解する
2. 該当箇所の原文を精読する
3. 著者のトーン・語彙を分析する
4. 方針に沿いつつ著者らしい文章で書き直す
5. 「なぜこう書いたか」をレポートに記載する

### 「削除」指示の場合
1. 削除の理由を確認する
2. 削除によって生じる論理的ギャップを埋めるブリッジ文を追加する
3. 削除した内容が他章で参照されていないか確認する

## 4. 著者トーンの維持チェック

リライト後、以下を確認する:

| チェック項目 | 基準 |
|-------------|------|
| 一人称 | 原文と同じ一人称を使っているか（「僕」「私」「俺」等） |
| 語尾パターン | 原文の語尾バリエーションを維持しているか |
| 口語レベル | 原文と同程度の口語表現を使っているか |
| 比喩・例え | 著者の比喩スタイルに合っているか |
| 読者への呼びかけ | 原文の呼びかけスタイルを維持しているか |

## 5. 章間整合チェック

リライトによって生じうる章間の不整合を防ぐ:

- **前方参照**: リライトした章で言及した内容が、前の章で矛盾していないか
- **後方参照**: 後の章で前提としている内容を変えていないか
- **数値の整合**: 利回り・経費率・金額等の数値が全章で一致しているか
- **用語の統一**: リライトで新たに導入した用語が他章でも統一されているか

## 6. 文字数の管理

- 新規追加の場合: 該当章の文字数予算内に収まるか確認
- 予算を超える場合: 冗長な箇所を削って調整するか、人間に相談
- 削除の場合: 削除分の文字数をレポートに記載
- 全体の文字数変動を章ごとに記録する
