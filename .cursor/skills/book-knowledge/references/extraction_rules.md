# ナレッジ抽出ルール

## 1. タグ定義と判定基準

### [主張]
- **定義:** 著者が「こうだ」「こうすべき」と断定・推奨している箇所
- **判定基準:** 動詞が断定形（「〜だ」「〜である」「〜すべき」「〜してはいけない」）
- **注意:** 事実の陳述と著者の意見を区別する

### [理由]
- **定義:** 主張を支える論理・根拠
- **判定基準:** 「なぜなら」「〜だから」「理由は」等の因果接続、または主張の直後に続く説明
- **注意:** 主張と理由は必ずペアで抽出する

### [具体例]
- **定義:** 主張を裏付ける実例・ケーススタディ
- **判定基準:** 固有名詞・具体的な状況描写・「例えば」で始まる箇所
- **注意:** 抽象的な例（「ある人が〜」）も含むが、具体性のレベルを記録する

### [数値]
- **定義:** 数字を伴うデータ・条件・計算
- **判定基準:** 数字が含まれる箇所すべてを候補とし、文脈上重要なものを抽出
- **記録形式:**
  ```
  [数値] 利回り8%以上が目安
  前提: 中古アパート、地方都市
  計算条件: 表面利回り（管理費・修繕費控除前）
  出典: L.42-45
  ```

### [反論]
- **定義:** 著者が想定する反論・批判とその反駁
- **判定基準:** 「〜と思うかもしれませんが」「よくある誤解ですが」等の構文
- **注意:** 著者自身の過去の誤解も含む

### [比喩]
- **定義:** 著者独自の比喩・たとえ話・アナロジー
- **判定基準:** 「〜のようなもの」「〜に例えると」、または直接的な比喩表現
- **保存:** 原文のまま記録（リライト不可）

### [体験談]
- **定義:** 著者自身の体験・エピソード
- **判定基準:** 一人称（「私は」「僕が」）で始まる過去の出来事
- **注意:** 顧客の事例と著者自身の体験を区別する

### [感情トーン]
- **定義:** その箇所の感情的な温度
- **種類:** 熱い主張 / 冷静な分析 / 共感的 / 警告的 / ユーモア / 後悔・反省 / 確信
- **判定基準:** 語気の強さ、感嘆符、修辞疑問文、副詞の強度

### [思想軸]
- **定義:** 著者の価値観・哲学が表出している箇所
- **判定基準:** 「大切なのは」「本質は」「信じている」等の価値判断を伴う表現
- **注意:** BookSpec の core_message との関連を明記する

## 2. セグメント分割ルール

### 分割のトリガー
- 話題の明確な切り替わり（「さて」「次に」「話は変わりますが」）
- テーマの転換（不動産投資→税金→融資 等）
- 質疑応答の開始/終了
- 動画/セクションの切り替わり

### セグメントの粒度
- 1セグメント = 1テーマ（原則）
- 大きすぎるテーマは「主張単位」で分割
- 目安: 1セグメントにつき主張1〜3個

## 3. 重複統合ルール

### 統合の判定基準
- 同一の主張が異なる箇所で繰り返されている
- 同一テーマについて異なる角度から語られている

### 統合時の処理
- 最も詳細な記述を「主記述」とする
- 他の箇所は「補足出典」として出典情報のみ記録
- ニュアンスが異なる場合は両方残す（差異を明記）

### 矛盾がある場合
- 両方の記述を `<!-- CONFLICT -->` タグ付きで残す
- 矛盾の概要と、どちらが新しい発言かを記録
- book-consistency（章間矛盾チェック）への入力となる

## 4. 章マッピングルール

toc.md が存在する場合:
- 各ナレッジ項目を最も関連性の高い章に紐付ける
- 複数章に関連する場合は全てに紐付け（`[関連章: 1, 3, 7]`）
- どの章にも当てはまらない項目は `[未分類]` とする

## 5. 品質基準

### 網羅性チェック
- cleaned_transcript.md の全セグメントが少なくとも1つのタグで抽出されている
- 著者の核心的な主張（BookSpec の core_message に関連）が漏れなく抽出されている

### 正確性チェック
- タグの分類が正確か（主張と理由の混同、体験談と具体例の混同がないか）
- 出典情報（行番号/タイムスタンプ）が正確か
- 著者の発言にない解釈が混入していないか
