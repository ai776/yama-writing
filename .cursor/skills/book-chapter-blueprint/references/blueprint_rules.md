# Blueprint 作成ルール

## 各フィールドの詳細ルールと記入例

フィールド定義は SKILL.md に記載済み。ここでは記入例と判定基準を補足する。

## 記入例: 不動産投資書籍

```markdown
# 第1章 Blueprint

### 章の核心（1文）
不動産だけが銀行レバレッジを使える投資である。

### 読者の現状認識
株やNISAは王道だと思っている。
不動産投資は怖い・自分には縁がないと感じている。

### 本章で壊す誤解
株＝最適解という思い込み。

### 本章のUSP
銀行融資というレバレッジ。他の投資商品では借入で投資できない。

### 優先順位ルール
1. USP提示（融資レバレッジの独自性）
2. 比較は補助的に（株との違いは背景説明程度）
3. 批判は最小限（株を否定するのではなく、不動産の強みを提示）

### 禁止事項
- 他商品をギャンブル呼ばわりしない
- 断定的な再現性保証をしない
- 「株はダメ」という論調にしない（「不動産には〜がある」に変換）
- 煽り口調にしない

### 必要要素
- 具体例1件（著者自身の初めての物件購入）
- 簡易比較図（自己資金100万円 × 株 vs 不動産のリターン比較）
- 初心者保護の一言（リスクも存在することの明示）
- K-001, K-003 のナレッジを使用

### 章のストーリー構造
①誤解解除 — 「株やNISAが最適」という常識を問い直す
②核心提示 — 銀行融資レバレッジは不動産だけの特権
③理由 — なぜ銀行は不動産に融資するのか（担保価値）
④具体例 — 著者の100万円が1,000万円の投資になった話
⑤行動含意 — 少額からでも始められるという安心感
⑥次章への橋渡し — 「では、どんな物件を選べばいいのか？」
```

## 章間整合チェックの基準

### USPの重複チェック
- 全章のUSPを並べて読み、同じ角度の章がないか確認
- 似ている場合は差別化ポイントを明記する

### 禁止事項の矛盾チェック
- 第1章で「批判は最小限」としたのに、第3章で「徹底比較」としていないか
- 全章の禁止事項が BookSpec の tone と一貫しているか

### ストーリー構造の多様性
- 全章が同じパターン（①誤解解除→②核心→...）にならないよう確認
- ハウツー章は「①現状→②手順→③注意点」等、章の性質に応じて変える
